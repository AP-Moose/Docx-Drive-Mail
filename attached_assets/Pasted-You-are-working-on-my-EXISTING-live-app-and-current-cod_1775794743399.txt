PRODUCT / UX CHANGES

- customer name

- optional project address

2. Guided voice capture section:

Use 5 guided prompts max:

Prompt 1:
“What does the customer want done?”

Prompt 2:
“What work is included?”

Prompt 3:
“Anything excluded, uncertain, or still to be decided?”

Prompt 4:
“What is the price or price range?”

Prompt 5:
“When can you start and how long should it take?”

3. Live proposal preview panel:

- Price
- Timeline
- Next Steps

====================
LIVE UPDATE LOGIC
====================

As the user speaks for each guided step:
- update the transcript for that step
- parse the step into the canonical proposal object
- immediately update the preview

Do this incrementally and simply.

For example:
- if user is on “What work is included?” and says three tasks, those should become bullets in “Work Included”
- if user mentions uncertainty, it should go into “Not Included / Assumptions”
- if user mentions price informally, normalize it into clean homeowner-friendly wording
- if user mentions a start time or duration, reflect it in Timeline

Do NOT over-engineer streaming intelligence if the stack does not support it cleanly.
A practical implementation is acceptable:
- live transcript visible immediately
- preview refreshes every few seconds / on pause / on step completion
That is good enough for V1.

====================
WRITING / OUTPUT QUALITY
====================

This is critical.

The preview and final proposal must sound like:
- a real local contractor
- clear
- trustworthy
- simple
- homeowner-friendly
- specific enough to feel real
- not generic AI mush
- not overly corporate

Rules:
- no “we are pleased to provide”
- no fluff
- no long bloated paragraphs
- use short sections and bullets
- do not hallucinate fake details
- if uncertain, say so clearly under assumptions/exclusions
- if price is rough, reflect it honestly
- preserve the contractor’s own words where appropriate, but clean them up

Proposal preview structure should be:

1. Proposal title
2. Summary of the job
3. Work included
4. Materials / fixtures if mentioned
5. Not included / assumptions
6. Price / payment terms
7. Timeline
8. Next steps

Make the preview useful even before every field is complete.
For example, if only first 2–3 prompts are done, still show a partial but well-structured draft.
